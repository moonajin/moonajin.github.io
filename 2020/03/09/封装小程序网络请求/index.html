封装小程序网络请求
瑾瑾 前端开发工程师
 2020-03-09  
  • 小程序 

提前准备

服务器域名配置

  • 每个微信小程序需要事先设置通讯域名，小程序只可以跟指定的域名进行网络通信。包括普通 HTTPS 请求（wx.request）、上传文件（wx.uploadFile）、下载文件（wx.downloadFile) 和 WebSocket 通信（wx.connectSocket）
  • 可以在「小程序后台-开发-开发管理-开发设置-服务器域名」中进行配置
    • 域名只支持 https 和 wss 协议
    • 域名不能使用 IP 地址（小程序的局域网 IP 除外）或 localhost
    • 可以配置端口，如 https://myserver.com:8080，但是配置后只能向 https://myserver.com:8080 发起请求。如果向 https://myserver.com、https://myserver.com:9091 等 URL 请求则会失败
    • 如果不配置端口。如 https://myserver.com，那么请求的 URL 中也不能包含端口，甚至是默认的 443 端口也不可以。如果向 https://myserver.com:443 请求则会失败
    • 域名必须经过 ICP 备案
    • 不支持配置父域名，使用子域名

目录结构

配置 config 文件

  • 大多数情况下，我们在实际开发中 API 都是部署在专用域名之下的，比如正式域名 https://api.example.com，测试域名 https://testapi.example.com 等，而后台给我们返回的接口地址通常是不带有域名的 /v1/example/list 这种格式，需要我们自己根据业务需要拼接出完整的请求路径。

  • 为了方便管理和后续维护，可以在 utils 文件夹下新建一个 config.js 文件，作为该项目的配置文件，用来存储所有项目相关的配置，比如 base url 以及 HTTP Header 等

封装请求

  • 在小程序中，我们通过 wx.request 方法来进行 HTTPS 请求，所以我们要对 wx.request 进行封装，wx.request的参数 url 是必填的，其余都是可选项
  • 我们可以把每个接口都需要的参数，进行封装，这样就不用每个接口都传一次了，也方便后续的维护，如果在某个接口中某一个通用参数的值是特殊的，那么可以直接在该接口中传新的参数值把旧的覆盖掉。
  • 默认情况下，我们会对 success 以及 fail 进行处理，如果想要手动处理异常，设置 isDealFail 的值为 true 就可以实现你想要的操作了

如何使用

  • 在 utils 文件夹下新建 api.js 文件，这里存放后台给我们的接口地址
  • 然后在需要的 js 文件中进行引用

其他

关于网络请求

  • 网络请求的默认超时时间和最大超时时间都是 60s

    • 超时时间可以在 app.json 中通过 networktimeout 进行配置

  • wx.request、wx.uploadFile、wx.downloadFile 的最大并发限制是 10 个

  • wx.connectSocket 的最大并发限制是 5 个

  • 小程序进入后台运行后，如果 5s 内网络请求没有结束，会回调错误信息 fail interrupted；在回到前台之前，网络请求接口调用都会无法调用

  • 只要成功接收到服务器返回，无论 statusCode 是多少，都会进入 success 回调
