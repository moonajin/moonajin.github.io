uniapp项目运行到Android模拟器
瑾瑾 前端开发工程师
 2022-04-11  
  • uniapp 

用 uniapp 进行 app 开发时，需要安装模拟器进行模拟，ios 模拟器的安装运行比较简单，按照官方文档的介绍很容易就能运行成功，可是官方文档对于 Android 模拟器的介绍只是一带而过，很多东西都不了解，因此走了很多弯路，所以记录下来以便参考，本文主要记录 Android 模拟器的适配。

环境版本

  • Android Studio：macOS 12.3.1
  • HbuilderX：3.3.13

一、Android Studio

1.下载并安装 Android Studio

  • 首先在官网下载并安装Android Studio

2.在 Android Studio 中安装模拟器

2.1 安装模拟器

  • 如需安装 Android 模拟器，可以在 preferencs 中搜索 sdk 关键词，找到 Android SDK ，在 Android SDK 的 SDK Tools 标签页中选择 Android Emulator 组件，点击 Apply 或 OK 进行下载，如有许可协议则同意所有许可协议即可。

2.2 安装官方推荐的软件包

2.2.1 SDK Tools 标签页下的工具
  • Android SDK 构建工具【必需】
    • 包含构建 Android 应用的工具，具体请参阅SDK 构建工具版本说明。
  • Android SDK 平台工具【必需】
    • 包含 Android 平台所需的各种工具，包括 adb 工具。
  • Android SDK 工具【必需】
    • 包括 ProGuard 等基本工具。请参阅 SDK 工具版本说明。
  • Android 模拟器【推荐】
    • 基于 QEMU 的设备模拟工具，可用于在实际的 Android 运行时环境中调试和测试应用。请参阅 Android 模拟器版本说明。
2.2.2 SDK Platforms 标签页下的工具
  • Android SDK Platform【必需】
    • 您的开发环境中必须至少有一个版本的 Android 平台，这样您才能构建应用。为了在最新设备上提供最佳用户体验，请使用最新版本的平台作为构建目标。您的应用仍然可以在旧版系统上运行，但您必须以最新版本为目标构建应用，以便在安装最新版 Android 的设备上运行应用时能够使用新功能。
  • Intel 或 ARM 系统映像【推荐】
    • 运行 Android 模拟器需要系统映像。每个版本的平台均包含受支持的系统映像。您也可以之后在从 AVD 管理器中创建 Android 虚拟设备 (AVD) 时下载系统映像。根据开发计算机的处理器选择 Intel 或 ARM。

3.在 Android Studio 中创建 Android 虚拟设备

  • 双击打开 Android Studio；

  • 在 Android Studio 欢迎屏幕上，依次选择 More Actions > Virtual Device Manager；

  • 点击 Create Device；

  • 随即会进入到 Select Hardware 窗口，选择所需的虚拟设备硬件配置，然后点击下一步 Next；

    • 如果未看到自己所需的硬件配置文件，可以创建或导入一个硬件配置文件

  • 接下来会进入到 System Image 窗口，选择所需的虚拟设备系统映像，如果您看到系统映像旁边的 Download，则需要点击以下载系统映像，不然无法进行下一步。若选择的系统映像不需下载，则可直接点击下一步 Next；

  • 然后会进入到 Android Virtual Device Configuration 窗口，可根据需要更改 AVD 属性，然后点击 Finish完成创建；

4.管理&运行虚拟设备

  • 完成后会回到管理虚拟设备的窗口，可以对虚拟设备进行管理；

  • 点击运行图标可运行对应的虚拟设备

    提示：当模拟器运行时，您将能够使用常见模拟器操作（如旋转），以及扩展控制选项（如导航播放）。如需在单独的窗口中运行模拟器，请依次转到 File > Settings > Tools > Emulator（在 macOS 设备中则依次转到 Android Studio > Preferences > Tools > Emulator），并取消选择 Launch in a tool window。

5.将 uniapp 项目运行到 Android 模拟器

  • uniapp 项目运行前，要先在 Android Studio 中启动 Android 模拟器，否则 uniapp 是检测不到模拟器的；

  • Android 模拟器启动后，HBuilderX 会将其识别为名称为 emulator-xxxx的 Android 手机；

  • 然后在 HbuilderX 中点击 运行 > 运行到手机或模拟器 > 对应的模拟器；
